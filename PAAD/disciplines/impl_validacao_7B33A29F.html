Discipline: Implementação e Testes Fase de Validação
Nesta fase, o foco principal da disciplina de Implementação é realizar os testes do produto em preparação para release, bem como ajustes pequenos com base no feedback do cliente.

Trata-se de uma disciplina intensa e plenamente relevante nesta fase do projeto.

Tasks
  • Planejar Testes Alfa em Ambiente de Desenvolvimento
  • Executar Testes Alfa em Build Estável no Ambiente de Desenvolvimento
  • Verificar se Necessita passar por Implementação ou se deve passar por Remodelagem
  • Implementar Correção de Erros
  • Iniciar Alterações de Modelagem em Disciplina de Análise e Projeto
  • Executar Builds Necessários, Integrar (caso necessário) e criar um Release
  • Gerar Relatório com Dados Coletados
Main Description

As atividades acontecem basicamente em dois momentos. No primeiro momento são planejados e executados testes no ambiente de desenvolvimento (testes alfa) e no segundo, o sistema é implantado e são executados testes de usabilidade e testes beta (teste no ambiente do cliente). Ao final, é gerado um relatório com as informações coletadas nos testes.

Encontrando erros após a execução dos testes alfa e/ou beta, dependendo de sua natuteza, a correção deverá ser feita na modelagem ou nos Códigos Fontes. No caso de uma remodelagem, as disciplinas de requisitos e análise e projeto são executadas e, ao final, realizam-se novamente os referidos testes.

Já no caso de problemas nos Códigos Fontes, uma nova execução da disciplina de implementação e testes deverá sanar o problema. São realizadas tantas iterações quantas forem necessárias, até que o uso do sistema seja aprovado pelo cliente.
