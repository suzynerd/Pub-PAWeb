Discipline: Gerência de Processo Fase de Construção
Durante a Construção, o Gerente trabalha pra manter o desenvolvimento do projeto dentro de uma margem aceitável do que foi planejado.

Essa fase é um ciclo de entregas de protótipos que, quando finalizada, apresenta um produto quase maduro para ser entregue.
Relationships

Tasks
  • Escrever Documento de Ambiente
  • Executar Reuniões com o Cliente ao Fim da Iteração com o Protótipo Desenvolvido
  • Registrar Reuniões em Ata de Reunião
  • Revisar Plano de Desenvolvimento de Software (PDS)
  • Desenvolver Plano de Iteração e Integração (PI&I)
  • Planejar Reuniões com o Cliente
Main Description

As atividades dentro dos focos são:

  • Ambiente

Encontra as incompatibilidades entre as ferramentas utilizadas e as trata. A construção utiliza o ambiente preparado nas duas fases anteriores e não acrescenta muito ao documento de ambiente.

  • Cliente

As reuniões com o cliente são feitas para entregas de protótipo. O principal artefato a ser entregue são os Códigos Fontes, então as reuniões devem ser acompanhadas por desenvolvedores capacitados (Implementador e Testador) para explicar em níveis de detalhe o que foi produzido. Da mesma forma, são os mais indicados para ouvirem os ajustes finos e rapidamente implementar as alterações desejadas.

  • Projeto

A principal atividade deste foco é integrar o que está sendo produzido a cada iteração. Possivelmente, muita informação nova será dada ao Gerente de Projetos, pois muitos casos de uso serão detalhados e mudanças podem surgir em todo o escopo do projeto. O ciclo de entregas deve ser planejado no Plano de Iteração e Integração e as revisões no Plano de Desenvolvimento de Software serão feitas à medida em que a implementação (ou não) dos casos de uso traga alterações ao cronograma.
