Activity: Gerência de Processo Validação
Durante a fase de validação, a disciplina de Gerência de Processo tem com objetivo entregar o Sistema de uma forma satisfatória, verificando sempre a aceitação do cliente como critério de finalização do projeto. Para isso, os trabalhadores cuidam para que o sistema funcione bem no ambiente onde será executado.

Description

Dentro dos três focos, as atividades são executadas da seguinte maneira:

  • Ambiente

O Gerente deve identificar os softwares necessários para que o sistema seja implantado e funcione normalmente, possivelmente num servidor do contratante. Após identificados, os softwares devem ser instalados e o sistema implantado. O Documento de Ambiente para esta fase relata o ambiente de implantação, e não mais de desenvolvimento.

  • Cliente

A(s) Reunião(ões) com o cliente entregam o sistema completo em funcionamento implantado em ambiente desejado por ele. Caso haja satisfação, o projeto é finalizado. Caso existam discordâncias, uma nova iteração é requerida para que sejam realizados os ajustes, essa nova iteração pode durar muito pouco, cerca de um ou dois dias. E assim sucessivamente, até que seja entregue o produto e o cliente esteja satisfeito. Se nas fases anteriores, o processo se desenvolveu como proposto, não existirão grandes alterações identificadas e requeridas pelo cliente, afinal de contas ele vem acompanhando desde cedo cada apresentação de protótipo. Por não existirem grandes mudanças a serem realizadas durante essa fase, o  Documento de Feedback é dispensável. 

  • Projeto

Após várias alterações no cronograma, durante esta fase pouco será alterado. O sistema já deve estar maduro e a finalização do projeto é dada como certa. As iterações finais já devem planejar, inclusive, uma comemoração para o fim do projeto.
