Activity: Requisitos Elaboração
O foco da Disciplina de Requisitos na Fase de Elaboração é o detalhamento inicial dos prováveis casos de uso que poderão ser utilizados para validar a arquitetura.

Description

Antes de inciar o detalhamento, são escolhidos os casos de uso que poderão ser desenvolvidos para validar a arquiterura. Tal escolha é feita levando-se em consideração a complexidade e a abragência do mesmo em relação à arquitetura. Mais de uma caso de uso é escolhido, pois o detalhamento desses casos de uso é quem vai definir qual deles é o mais adequado para realizar a validação da arquitetura.

Após escolhidos os casos de uso, é realizado o detalhamento inicial dos mesmos. Será confeccionado o Documento de Especificação de Caso de Uso, que conterá as informações necessárias para que a próxima disciplina, Análise e Projeto, possa ser realizada com sucesso.

Caso venham a surgir dúvidas, inseguranças ou inconsistências no detalhamento, o cliente será entrevistado ou consultado por algum meio de comunicação. Após sanar a dúvida, realiza-se novamente a primeira atividade, ajustando o que for necessário.

Após detalhar o caso de uso, é feita uma revisão nos requisitos, o que pode ocasionar o surgimento de um novo caso de uso, implicando em uma revisão no Diagrama Geral de Casos de Uso (UML) e também no Documento de Visão. Concluídas todas as atividades, inicia-se a disciplina de Análise e Projeto.
